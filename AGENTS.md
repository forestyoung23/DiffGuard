# Agent Instructions

## Changelog

For every user-visible change, update `CHANGELOG.md` under `Unreleased`.

Use these categories when they apply:
- `Added`
- `Changed`
- `Fixed`
- `Removed`

Do not add changelog entries for internal-only refactors, test-only changes, formatting, or build cleanup unless they affect users.

When preparing a release, move `Unreleased` entries under the released version and date.
