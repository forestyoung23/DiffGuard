# DiffGuard V1.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Improve first-release usability with richer findings, provider presets, connection testing, and result-panel actions.

**Architecture:** Keep the current review pipeline intact. Add optional metadata to findings, keep JSON parsing backward compatible, introduce small settings/provider helpers, and extend the existing Swing result panel with filter/copy/rerun callbacks.

**Tech Stack:** Kotlin, IntelliJ Platform Swing UI DSL, kotlinx.serialization, OkHttp, JUnit 5.
